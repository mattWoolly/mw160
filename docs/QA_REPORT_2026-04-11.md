## 5. Top 5 things to fix first

Ordered by (a) severity, (b) how much they gate the rest of the work:

1. **`QA-CONF-001`: Bool parameter state restoration.**
   `src/PluginProcessor.cpp:217-222`. Reproducible under pluginval
   @ strictness 10 (3/3 seeds). **Every CI run will be red until this
   is fixed**, so it gates the entire CI baseline. The fix is tightly
   scoped: in `setStateInformation`, after `replaceState`, iterate each
   bool parameter and explicitly re-apply via `setValueNotifyingHost`.
   Pair with a DAW-style round-trip unit test (`QA-CONF-006` /
   `QA-DSP-007` both benefit from the same new test fixture).
   --- CRITICAL via its linked `QA-CI-002` entry.

2. **`QA-UX-001`: Threshold labeled `dBu` but DSP operates in `dBFS`.**
   `src/PluginProcessor.cpp:31`; `src/dsp/Compressor.cpp:74-80`. The
   ~18 dB semantic mismatch means every factory preset is off by that
   amount relative to user intuition, every threshold LED comparator
   inherits the lie (`QA-UX-006`), and every parameter automation
   drawn into a DAW is misaligned with the knob label. Recommended
   resolution: change the label to `dBFS` (smaller change, no stateful
   calibration). Rebalance factory presets in the same PR. Required
   before the plugin can be called "usable out of the box."
   --- CRITICAL.

3. **`QA-UX-002`: Ratio doesn't actually reach infinity.**
   `src/dsp/GainComputer.cpp:14-23`. `slope = 1/R - 1` gives `-0.9833`
   at `R=60`, not `-1.0`. The "Brick Wall" preset is not actually a
   brick wall. One-branch fix in `GainComputer::computeGainReduction`
   that short-circuits to `slope = -1.0` when `ratio >= 50`. Required
   because "the ratio control must reach infinity" is an
   identity-defining property of the hardware class per
   REFERENCE.md §2.1. --- CRITICAL.

4. **`QA-DSP-001`: ParameterSmoother restarts its ramp every block →
   block-size-dependent output.** `src/dsp/ParameterSmoother.h:36-55`.
   Output deviates by up to 1.7 dB for block sizes < 1024 samples at
   48 kHz. Masked at large block sizes, so a DAW user changing buffer
   size will hear a subtle tone shift during parameter automation.
   One-line fix: early-out unconditionally when
   `newTarget == target_`. Closely related to the `-Wfloat-equal`
   warning the build emits at that line; both symptoms collapse into
   one fix. --- HIGH, and a precondition for the bypass crossfade
   (`QA-DSP-003` / `QA-UX-003` / `DESIGN-IMPL-001`) to be block-size
   independent when it is added.

5. **Bypass parameter (triple-filed HIGH: `QA-DSP-003` + `QA-UX-003`
   + `DESIGN-IMPL-001`).** `src/PluginProcessor.cpp` (add
   `bypass` APVTS bool, override `getBypassParameter()`),
   `src/PluginEditor.cpp` (attach the already-drawn BYPASS pill),
   `processBlock` (crossfade wet→dry over 20 ms on change, using the
   existing `ParameterSmoother`). Once `QA-DSP-001` is fixed this
   crossfade behaves correctly across block sizes. Required by
   REFERENCE.md §2.2 and by the rendered editor (the pill is already
   drawn --- it just toggles a local visual state with no effect on
   audio).

**Runner-up (optional but easy and visible):**
- **`DESIGN-REVIEW-006` / `QA-UX-004`:** one-line fix of the
  `"OverEasy"` display-name string in `src/PluginProcessor.cpp:55` to
  `"Knee"`. Keeps the ID intact (no state-migration risk) and
  eliminates the most user-visible brand leak in the project
  immediately, without waiting for the full `QA-TM-001` scrub.

## 7. Risks and open questions surfaced by this pass

1. **`QA-DSP-004` --- the implementation is feedforward, REFERENCE.md
   §1.1 calls for feedback.** This is a design-level identity
   question, not a code bug. Resolving requires either a topology
   rewrite (re-wire the detector from the post-processing signal with
   a one-sample delay) or a documented exception in REFERENCE.md §1.1.
   The topology rewrite is not trivial --- it interacts with stereo
   linking, the existing ballistics, and the static-curve test
   fixtures. Team decision needed before a dev-agent picks this up.
2. **Migration coupling between `QA-TM-001` and `QA-CONF-005`.** The
   trademark scrub of parameter ID `overEasy`→`softKnee` will break
   every existing user preset unless the migration shim
   (`QA-CONF-005`) lands in the same PR. These two must not be
   decoupled.
3. **Factory preset re-balance.** `QA-UX-001`'s resolution requires
   re-dialling all seven factory presets against the new threshold
   convention. This is additional scope on top of the unit-label
   change.
4. **`QA-DSP-007`: plugin-processBlock-layer test fixture is missing.**
   Several HIGH findings (`QA-DSP-001`, `QA-DSP-003`, `QA-CONF-001`,
   `QA-CONF-006`) benefit from the same new test fixture that
   instantiates `MW160Processor` and runs `processBlock` in a Catch2
   case. Suggest landing the fixture first as its own change, then
   using it to drive the fixes for the HIGH findings.
5. **Font binaries (`DESIGN-IMPL-002`).** Inter and JetBrains Mono
   were not available to check into the repo in this pass. Several
   LOW / MEDIUM findings (`DESIGN-REVIEW-002` wordmark overlap,
   `DESIGN-REVIEW-003` tick column narrow) are harder to evaluate
   against the JUCE fallback font than they will be against Inter.
   Fix order: embed fonts first, then re-evaluate the layout drift.
